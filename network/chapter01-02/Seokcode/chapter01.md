# Chapter 01. 인터넷이란?

## 1. 인터넷이란?

### 1.1 하드웨어와 소프트웨어 관점

- **인터넷**
  전 세계적으로 수십억 개의 컴퓨팅 장치를 연결하는 컴퓨터 네트워크이다.

- **호스트(Host), 종단 시스템(End System)**
  인터넷에 연결된 컴퓨팅 장치이며, 역할에 따라 **클라이언트(Client)** 와 **서버(Server)** 로 구분된다.

- **통신 링크(Communication Link)**
  다양한 전송률(bps)로 패킷(데이터)을 전달하는 매체이다.
  예: 동축 케이블, 구리선

- **패킷(Packet)**
  데이터를 세그먼트로 나누고 헤더를 붙여 목적지까지 전달한 뒤, 목적지에서 다시 조립하는 데이터 단위이다.

- **패킷 교환기(Packet Switch)**
  목적지 방향으로 패킷을 전달하기 위해 사용되는 장치이다.
  - **스위치(Switch)**: 접속 네트워크에서 사용
  - **라우터(Router)**: 네트워크 코어에서 사용

- **경로(Path)**
  시작 지점부터 목적지까지 도달하는 동안 거치는 통신 링크와 패킷 교환기의 집합이다.

- **ISP(Internet Service Provider)**
  패킷 스위치와 통신 링크로 이루어진 네트워크이다. 종단 시스템에 다양한 네트워크 접속을 제공하며, 인터넷은 종단 시스템들을 서로 연결하는 구조이므로 ISP들 또한 서로 연결되어 있어야 한다.

  하위 계층 ISP는 국가 및 국제 상위 계층 ISP를 통해 서로 연결되고, 상위 계층 ISP들은 직접 연결된다. 각 ISP 네트워크는 별도로 관리되며, IP 프로토콜을 수행하고 네이밍 및 주소 배정 방식을 따른다.

# 이미지: ISP

- **프로토콜(Protocol)**
  정보의 송수신을 위한 통신 규약이다.
  예: TCP/IP

  서로 다른 프로토콜을 사용하면 상호작용할 수 없다.

# 이미지: Protocol

- **네트워크 프로토콜**
  둘 이상의 개체가 포함된 인터넷 환경에서 일어나는 모든 통신 활동을 제어한다.

  예시:
  - **혼잡 제어(Congestion Control)**: 종단 시스템에 존재하며, 송수신자 간에 전송되는 패킷 전송률을 조절한다.
  - **라우터에서의 프로토콜**: 출발지에서 목적지까지 패킷이 이동할 경로를 설정한다.

### 1.2 서비스 측면에서 본 인터넷

인터넷은 애플리케이션에 서비스를 제공하는 **인프라스트럭처(Infrastructure)** 이다.

- **소켓 인터페이스(Socket Interface)**
  한 종단 시스템에서 수행되는 프로그램이 다른 종단 시스템에서 수행되는 특정 목적지 프로그램으로 데이터를 전달하도록, 인터넷 인프라스트럭처에 어떤 방식으로 요청해야 하는지를 명시한 인터페이스를 의미한다.

### 1.3 네트워크 코어

링크와 스위치의 네트워크를 통해 데이터를 이동시키는 방식은 크게 두 가지이다.

1. **패킷 교환(Packet Switching)**: 자원을 미리 보장하지 않는다.
2. **회선 교환(Circuit Switching)**: 자원을 예약하므로 보장된다.

#### 1.3.1 패킷 교환

종단 시스템들은 서로 메시지를 교환한다.

1. 송신 시스템은 메시지를 **패킷** 이라는 데이터 덩어리로 분할한다.
2. 각 패킷은 통신 링크와 패킷 스위치를 거친다.
3. 각 패킷은 링크의 최대 전송률에 맞는 속도로 각 통신 링크에서 전송된다.

패킷 스위치에는 라우터와 링크 계층 스위치가 존재한다.

#### 저장 후 전달(Store-and-Forward) 전송 방식

스위치가 패킷의 첫 비트를 출력 링크로 전송하기 전에, 전체 패킷을 먼저 모두 받아야 한다.

# 이미지: 저장 후 전달 전송 방식

출발지는 목적지로 전송할 3개의 패킷(1, 2, 3)을 가지고 있으며, 각 패킷은 `L` 비트로 구성되어 있다. 출발지는 링크에서 `L` 비트의 패킷을 `R bps(bits per second)`의 속도로 송신한다.

이때 경과 시간은 다음과 같이 정리할 수 있다.

- `0초`: 출발지가 패킷 1을 전송하기 시작한다.
- `L/R초`: 라우터는 패킷 1을 수신 완료하고, 이를 전송하기 시작한다. 동시에 출발지는 패킷 2를 전송하기 시작한다.
- `2L/R초`: 라우터는 패킷 2를 수신 완료하고, 이를 전송하기 시작한다. 동시에 목적지는 패킷 1 전체를 수신 완료하고, 출발지는 패킷 3을 전송하기 시작한다.
- `3L/R초`: 라우터는 패킷 3을 수신 완료하고, 이를 전송하기 시작한다. 동시에 목적지는 패킷 2 전체를 수신 완료한다.
- `4L/R초`: 목적지는 패킷 3 전체를 수신 완료한다.

따라서 저장 후 전달 전송 방식을 채택한다면, 목적지는 `4L/R초`에 3개의 모든 패킷을 수신하게 된다.

#### 큐잉 지연

패킷 스위치는 여러 링크에 연결되어 있고, 각 링크에 대해 출력 버퍼를 가진다. 출력 큐는 해당 링크로 송신하려는 패킷들을 임시로 저장하는 공간이다.

도착한 패킷이 특정 링크로 전송되어야 하지만 그 링크가 이미 다른 패킷을 전송 중이라면, 새로 도착한 패킷은 출력 큐에서 대기해야 한다. 이때 발생하는 지연이 **큐잉 지연(Queuing Delay)** 이다.

- 큐잉 지연은 가변적이며, 네트워크 혼잡 정도에 따라 달라진다.
- 큐의 공간은 유한하므로 **패킷 손실(Packet Loss)** 이 발생할 수 있다.
- 큐가 가득 찬 상태에서 패킷이 도착하면, 도착 패킷 또는 대기 중인 패킷 중 일부를 폐기해야 할 수 있다.

#### 포워딩 테이블과 라우팅 프로토콜

라우터는 한 링크로 도착한 패킷을 다른 링크로 전달한다. 이때 핵심은 "어떤 링크로 보낼 것인가"를 결정하는 것이다.

##### IP 주소

출발지는 패킷 헤더에 목적지 IP 주소를 포함한다.

##### 포워딩 테이블

각 라우터는 목적지 주소 또는 목적지 주소의 일부를 출력 링크에 매핑하는 **포워딩 테이블(Forwarding Table)** 을 가진다.

동작 과정은 다음과 같다.

1. 패킷이 라우터에 도착한다.
2. 라우터는 패킷의 IP 주소를 조사한다.
3. 해당 목적지 주소를 이용해 포워딩 테이블을 검색한다.
4. 검색 결과에 따라 패킷을 적절한 출력 링크로 보낸다.

이 포워딩 테이블은 특별한 **라우팅 프로토콜(Routing Protocol)** 을 통해 자동으로 설정된다.
예: 각 라우터로부터 각 목적지까지의 최단 경로 계산

#### 회선 교환

1. 송신자가 정보를 보내기 전에, 네트워크는 송신자와 수신자 사이의 연결을 먼저 설정해야 한다.
2. 회선을 설정할 때, 연결이 유지되는 동안 네트워크 링크의 일정 전송률을 예약한다.
3. 전송률이 예약되어 있으므로 송신자는 수신자에게 보장된 전송률로 데이터를 보낼 수 있다.

##### 회선 교환 네트워크에서의 다중화

1. **주파수 분할 다중화(FDM)**
   링크가 연결되는 동안 각 연결에 대해 주파수 대역을 고정으로 제공한다.

2. **시분할 다중화(TDM)**
   시간을 일정 주기의 프레임으로 구분하고, 각 프레임을 시간 슬롯으로 나누어 슬롯을 연결에 할당한다.

##### 패킷 교환과 회선 교환 비교

- **회선 교환 방식**: 요구와 관계없이 미리 전송 링크 사용량을 할당한다.
- **패킷 교환 방식**: 필요할 때만 링크 사용량을 할당한다.

일반적으로 패킷 교환 방식이 회선 교환 방식보다 자원 활용 측면에서 더 효율적이다.

#### 네트워크 구조

인터넷 구조는 단순한 단일망이 아니라, 여러 ISP와 계층 구조가 결합된 형태로 발전해왔다.

1. 모든 접속 ISP를 하나의 글로벌 통과 ISP에 연결하는 구조
2. 수십만 개의 접속 ISP와 다수의 글로벌 ISP, 지역 ISP가 함께 존재하는 구조
3. 다중 계층 구조
   예: 접속 ISP, 지역 ISP, 1계층 ISP
4. 다중 계층 구조 + PoP + 멀티홈 + 피어링 + IXP
   `IXP(Internet Exchange Point)` 는 여러 ISP가 서로 피어링할 수 있는 접속 지점이다.
5. 다중 계층 구조 + PoP + 멀티홈 + 피어링 + IXP + 콘텐츠 제공 네트워크

오늘날의 인터넷은 "네트워크의 네트워크"라고 할 수 있으며, 소수의 1계층 ISP와 매우 많은 하위 계층 ISP들로 구성된다.

#### 패킷 교환 네트워크에서의 지연

패킷은 출발지에서 목적지까지 이동하면서 여러 라우터를 거친다. 이 과정에서 각 노드마다 다양한 지연을 겪으며, 이것들이 누적되어 전체 지연이 된다.

- **처리 지연(Processing Delay)**
  패킷 헤더를 검사하고, 그 패킷을 어디로 보낼지 결정하는 시간이다.

- **큐잉 지연(Queuing Delay)**
  패킷이 큐에서 링크로 전송되기를 기다리는 시간이다.

- **전송 지연(Transmission Delay)**
  패킷의 모든 비트를 링크로 밀어 넣는 데 필요한 시간이다.

- **전파 지연(Propagation Delay)**
  비트가 한 라우터에서 다음 라우터까지 링크를 따라 이동하는 데 필요한 시간이다.

전체 노드 지연은 다음과 같이 표현할 수 있다.

```text
전체 노드 지연 = 처리 지연 + 큐잉 지연 + 전송 지연 + 전파 지연
```

#### 큐잉 지연을 결정하는 주요 요소

- 트래픽이 큐에 도착하는 비율
- 링크의 전송률
- 도착하는 트래픽의 특성

관련 변수는 다음과 같다.

- 패킷 도착 평균율: `a` 패킷/초
- 전송률: `R` 비트/초
- 각 패킷 길이: `L` 비트

트래픽 강도는 다음과 같이 표현한다.

```text
La / R
```

- `La/R > 1`
  평균 도착 비율이 처리 가능한 비율보다 크므로 큐가 무한히 증가한다.

- `La/R < 1`
  큐잉 지연은 트래픽 특성에 따라 달라진다.

# 이미지: 트래픽 강도와 큐잉 지연

큐 공간이 무한하지 않다면, 트래픽 강도가 1에 가까워질수록 큐가 자주 가득 차게 된다. 이때 더 이상 패킷을 저장할 수 없으면 라우터는 해당 패킷을 폐기하며, 손실된 패킷은 종단 간 재전송될 수 있다.

#### 네트워크 성능 측정 요소

- 지연 정도
- 패킷 손실 확률

### 1.4 인터넷 프로토콜 스택

인터넷 프로토콜 스택은 일반적으로 **5계층** 구조로 설명한다.

# 이미지: 인터넷 프로토콜 스택

#### 애플리케이션 계층

- 프로토콜 예시: `HTTP`, `SMTP`, `FTP`
- `DNS` 역시 애플리케이션 계층에 존재한다.

#### 트랜스포트 계층

클라이언트와 서버 간에 애플리케이션 계층 메시지를 전송하는 서비스를 제공한다.

- 데이터 단위: `Segment`
- 주요 프로토콜:
  1. **TCP**
     연결 지향형 서비스를 제공하며, 전달 보장, 흐름 제어, 혼잡 제어 기능을 제공한다.
  2. **UDP**
     비연결형 서비스를 제공하며, 전달 보장, 흐름 제어, 혼잡 제어 기능을 제공하지 않는다.

#### 네트워크 계층(IP 계층)

한 호스트에서 다른 호스트로 **데이터그램(Datagram)** 을 라우팅하는 책임을 가진다.

- 데이터 단위: `Datagram`
- 대표 프로토콜: `IP`

인터넷의 네트워크 계층은 라우팅 프로토콜도 포함한다.

##### 네트워크의 네트워크

인터넷은 여러 개별 네트워크가 연결된 구조이다.

```text
[집 네트워크]
        ├─ [통신사 네트워크] ─ [해외 백본망] ─ [구글 네트워크]
[학교 네트워크]
[회사 네트워크]
```

라우팅 프로토콜은 출발지와 목적지 사이에서 데이터그램이 이동할 경로를 결정한다. 인터넷 전체는 공통적으로 IP를 사용하지만, 각 네트워크 내부에서 경로를 찾는 방식은 서로 다를 수 있다.

#### 링크 계층

전체 프레임을 한 네트워크 요소에서 인접한 네트워크 요소로 이동시킨다.

- 데이터 단위: `Frame`

각 노드에서 네트워크 계층은 데이터그램을 아래 링크 계층으로 보내고, 링크 계층은 이를 다음 노드로 전달한다. 다음 노드에 도착하면 다시 상위 네트워크 계층으로 전달된다.

링크 계층이 제공하는 서비스는 해당 링크에서 채택한 링크 계층 프로토콜에 의해 결정된다.

예:

- 노트북 ↔ 공유기: `Wi-Fi` 프로토콜
- 공유기 ↔ 통신사 장비: `Ethernet` 프로토콜

#### 캡슐화

# 이미지: 캡슐화

각 계층에서 패킷은 **헤더 필드(Header Field)** 와 **페이로드 필드(Payload Field)** 를 가진다. 한 계층의 페이로드는 일반적으로 상위 계층에서 내려온 데이터를 의미하며, 계층이 내려갈수록 각 계층의 헤더가 덧붙는 방식으로 캡슐화가 이루어진다.
