## 3.1 트랜스포트 계층

### 핵심 개념
- 트랜스포트 계층 → 프로세스 간 통신 제공
- 네트워크 계층 → 호스트 간 통신 제공

> 트랜스포트는 “프로세스 단위”, 네트워크는 “호스트 단위”

---

### 논리적 통신 (Logical Communication)
- 실제로는 네트워크를 거치지만
- 애플리케이션 입장에서는 직접 연결된 것처럼 보임

---

### 캡슐화 흐름 (정리 추가)
Application → Segment → Datagram → Frame

1. Application 메시지
2. Transport에서 segment 생성
3. Network에서 datagram
4. Link에서 frame

---

### 트랜스포트 계층 역할 (추가 핵심)
- Segmentation (메시지 분할)
- Multiplexing / Demultiplexing
- Error detection (오류 검출)
- (TCP의 경우) 신뢰성 제공

---

## 3.1.1 트랜스포트 계층 vs 네트워크 계층

### 차이 핵심 (중요)
- Transport → 프로세스 간 통신
- Network → 호스트 간 통신

---

### 제약 관계 (추가 핵심)
- 트랜스포트 계층은 네트워크 계층 위에서 동작
- 네트워크가 제공하지 않는 기능은 직접 구현해야 함

예:
- IP는 비신뢰적
→ TCP가 신뢰성 제공

---

## 3.1.2 인터넷 트랜스포트 계층 개요

### TCP
- 신뢰성 제공
- 연결 지향
- 흐름 제어
- 혼잡 제어

### UDP
- 비신뢰적
- 비연결형
- 빠름
- 최소 기능

---

### IP (중요 추가)
> best-effort delivery

- 전달 보장 없음
- 순서 보장 없음
- 데이터 무결성 보장 없음

→ 완전 비신뢰적

---

## 세그먼트 vs 데이터그램 (명확화)

- Segment → 트랜스포트 계층 단위
- Datagram → 네트워크 계층 단위

---

## 3.2 다중화와 역다중화

### 핵심 개념 (중요)
> 호스트 → 프로세스 전달 확장

---

### Multiplexing
> 여러 소켓의 데이터를 하나로 모아 전송

- 송신 측 동작

---

### Demultiplexing
> 수신된 데이터를 올바른 소켓으로 전달

- 수신 측 동작

---

## Port 번호 (핵심 강화)
- 각 소켓은 고유 포트 번호
- 목적지 포트 번호로 프로세스 식별

---

## UDP 다중화 특징 (추가 핵심)
- 기준: (목적지 IP, 목적지 Port)
- 같은 목적지면 같은 프로세스로 전달

---

## TCP 다중화 특징 (중요)
> 4-tuple 기반

- 출발지 IP
- 출발지 Port
- 목적지 IP
- 목적지 Port

→ 이 4개로 연결 식별

---

## TCP 연결 구조 (핵심 보강)

### 서버 구조
- welcome socket (리스닝)
- connection socket (연결별 생성)

---

### 특징
- 하나의 서버가 여러 클라이언트 처리 가능
- 각 연결은 독립적인 소켓

---

## 웹 서버 구조 (추가 핵심)
- 멀티 프로세스 or 멀티 스레드 방식
- 최근: 멀티 스레드 + 이벤트 기반

---

## HTTP와 TCP 연결 관계 (정리 강화)

### Persistent
- 하나의 TCP 연결 유지

### Non-persistent
- 요청마다 TCP 생성

---
