1. TCP와 UDP의 차이는 무엇인가요?

답변

TCP는 연결 지향형 프로토콜이고, UDP는 비연결형 프로토콜입니다.

TCP는 데이터를 보내기 전에 연결을 맺고, 데이터가 제대로 도착했는지 확인하면서 전송합니다. 그래서 신뢰성이 높지만 상대적으로 속도가 느립니다. 예를 들어 웹 통신, 파일 전송, 이메일처럼 데이터가 정확히 전달되어야 하는 경우에 사용됩니다.

반면 UDP는 연결 과정 없이 데이터를 바로 보냅니다. 도착 여부나 순서를 보장하지 않기 때문에 신뢰성은 낮지만 속도가 빠릅니다. 그래서 실시간성이 중요한 영상 스트리밍, 온라인 게임, 음성 통화 등에 사용됩니다.

2. 쿠키, 세션, JWT의 차이는 무엇인가요?

답변

쿠키는 클라이언트인 브라우저에 저장되는 작은 데이터입니다. 서버가 응답할 때 쿠키를 내려주면, 브라우저는 이후 요청마다 해당 쿠키를 함께 보냅니다.

세션은 로그인 상태 같은 사용자 정보를 서버에 저장하는 방식입니다. 클라이언트는 세션 ID만 쿠키로 가지고 있고, 실제 사용자 정보는 서버에 저장됩니다. 보안 측면에서는 유리하지만, 사용자가 많아지면 서버 메모리나 세션 저장소 관리가 필요합니다.

JWT는 인증 정보를 토큰 형태로 만들어 클라이언트가 저장하고 요청마다 전달하는 방식입니다. 서버가 세션을 직접 저장하지 않아도 되기 때문에 확장성에 장점이 있습니다. 다만 토큰이 탈취되면 만료 전까지 악용될 수 있어서 만료 시간 관리와 HTTPS 사용이 중요합니다.

3. OSI 7계층을 말해보세요.

답변

물리 계층 → 데이터 링크 계층 → 네트워크 계층 → 전송 계층 → 세션 계층 → 표현 계층 → 응용 계층입니다.
1계층인 물리 계층은 전기 신호나 케이블, 무선 신호처럼 실제 데이터를 전송하는 물리적인 부분을 담당합니다.

2계층인 데이터 링크 계층은 같은 네트워크 안에서 MAC 주소를 이용해 데이터를 전달합니다.

3계층인 네트워크 계층은 IP 주소를 이용해 목적지까지 가는 경로를 찾습니다.

4계층인 전송 계층은 TCP나 UDP를 이용해서 데이터가 안정적으로 전달되도록 관리합니다.

5계층인 세션 계층은 통신 연결을 설정하고 유지하고 종료하는 역할을 합니다.

6계층인 표현 계층은 데이터 형식 변환, 암호화, 압축 등을 담당합니다.

7계층인 응용 계층은 사용자가 직접 사용하는 서비스와 가장 가까운 계층으로, HTTP, FTP, SMTP 같은 프로토콜이 여기에 해당합니다.

4. 캡슐화와 역캡슐화가 무엇인가요?

답변

캡슐화는 데이터를 보낼 때 상위 계층의 데이터에 하위 계층의 헤더를 붙여 내려가는 과정입니다.

예를 들어 응용 계층에서 만든 데이터가 전송 계층으로 내려가면 TCP 헤더가 붙고, 네트워크 계층에서는 IP 헤더가 붙고, 데이터 링크 계층에서는 MAC 주소 정보가 붙습니다.

반대로 수신 측에서는 각 계층을 올라가면서 붙어 있던 헤더를 하나씩 제거하고 원래 데이터를 복원합니다. 이 과정을 역캡슐화라고 합니다.

5. 각 계층에서 사용하는 데이터 단위는 무엇인가요?

답변

계층마다 데이터를 부르는 단위가 다릅니다.

응용, 표현, 세션 계층에서는 보통 데이터라고 부릅니다.

전송 계층에서는 세그먼트라고 부릅니다. 다만 UDP의 경우에는 데이터그램이라고도 합니다.

네트워크 계층에서는 패킷이라고 부릅니다.

데이터 링크 계층에서는 프레임이라고 부릅니다.

물리 계층에서는 비트 단위로 전송됩니다.

6. 스위치와 라우터는 각각 몇 계층 장비인가요?

답변

스위치는 보통 2계층 장비입니다. MAC 주소를 기반으로 같은 네트워크 내부에서 데이터를 전달합니다.

라우터는 3계층 장비입니다. IP 주소를 기반으로 서로 다른 네트워크 간에 데이터를 전달하고, 목적지까지 가는 경로를 선택합니다.

그래서 스위치는 같은 LAN 내부 통신에 주로 사용되고, 라우터는 네트워크와 네트워크를 연결할 때 사용됩니다.

7. IP는 OSI 몇 계층에 해당하나요?

답변

IP는 OSI 7계층 중 3계층인 네트워크 계층에 해당합니다.

IP는 출발지와 목적지의 IP 주소를 기반으로 데이터를 어느 경로로 보낼지 결정하는 데 사용됩니다.

즉, IP는 서로 다른 네트워크를 지나 목적지까지 데이터를 전달하는 역할을 합니다.

8. HTTP는 OSI 몇 계층에 해당하나요?

답변

HTTP는 OSI 7계층 중 7계층인 응용 계층에 해당합니다.

HTTP는 웹 브라우저와 웹 서버가 데이터를 주고받기 위한 프로토콜입니다.

예를 들어 사용자가 웹 페이지에 접속하면 브라우저는 HTTP 요청을 보내고, 서버는 HTML, JSON, 이미지 같은 데이터를 HTTP 응답으로 반환합니다.

9. 같은 네트워크 안에서 통신할 때와 다른 네트워크로 통신할 때 차이는 무엇인가요?

답변

같은 네트워크 안에서 통신할 때는 ARP를 통해 목적지 IP의 MAC 주소를 알아낸 뒤 직접 데이터를 보냅니다.

반면 다른 네트워크로 통신할 때는 목적지 장비에 직접 보낼 수 없기 때문에 기본 게이트웨이, 즉 라우터의 MAC 주소를 찾아서 라우터로 데이터를 보냅니다.

라우터는 IP 주소를 보고 다음 경로로 패킷을 전달합니다.

10. ARP는 무엇인가요?

답변

ARP는 IP 주소를 MAC 주소로 변환하기 위한 프로토콜입니다.

통신할 대상의 IP 주소는 알고 있지만, 실제 같은 네트워크 안에서 데이터를 보내려면 MAC 주소가 필요합니다. 이때 ARP 요청을 보내서 해당 IP를 가진 장비의 MAC 주소를 알아냅니다.

예를 들어 같은 LAN 안에서 특정 IP로 데이터를 보내려면 먼저 ARP를 통해 그 IP에 해당하는 MAC 주소를 찾습니다.
